Cascading Style Sheets

The language we use to style and layout Web pages, transforming HTML into visually engaging experiences.

What is CSS?

CSS (Cascading Style Sheets) is used to style and layout web pages — from altering colors and fonts to creating complex responsive layouts.

Visual Styling

Control colors, fonts, spacing, and other visual aspects of web content.

Layout Control

Create flexible layouts with Flexbox, Grid, and positioning properties.

Responsive Design

Adapt your designs to different screen sizes with media queries.

CSS Code Examples

Basic CSS Syntax

/* Select element by tag name */
body {
  background-color: #0a192f;
  color: #e6f1ff;
  font-family: 'Montserrat', sans-serif;
}

/* Select element by class */
.card {
  background-color: #112240;
  border-radius: 10px;
  padding: 20px;
  box-shadow: 0 5px 15px rgba(0, 0, 0, 0.1);
}

/* Select element by ID */
#header {
  text-align: center;
  margin-bottom: 2rem;
}

/* Pseudo-class example */
a:hover {
  color: #64ffda;
  text-decoration: none;
}

CSS Flexbox & Grid

/* Flexbox container */
.container {
  display: flex;
  justify-content: space-between;
  align-items: center;
  flex-wrap: wrap;
}

/* CSS Grid layout */
.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

/* Media query for responsiveness */
@media (max-width: 768px) {
  .grid-container {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* CSS variables (custom properties) */
:root {
  --primary-color: #64ffda;
  --card-bg: #112240;
}

.element {
  color: var(--primary-color);
  background-color: var(--card-bg);
}

Evolution of CSS

1996

CSS Level 1

The first CSS specification published by W3C, providing basic font, color, and spacing properties.

1998

CSS Level 2

Added positioning, z-index, media types, and bidirectional text support.

2011

CSS 2.1

Cleaned up CSS2 specification, removing poorly supported features.

Ongoing

CSS3

Modular approach with media queries, namespaces, selectors, flexbox, grid, animations and more.

CSS Best Practices

Do's

  • Use semantic class names
  • Organize CSS with a methodology (BEM, SMACSS)
  • Use CSS variables for theming
  • Write mobile-first responsive code
  • Use flexbox/grid for layouts

Don'ts

  • Don't use !important unnecessarily
  • Avoid overly specific selectors
  • Don't use inline styles
  • Avoid deep nesting in preprocessors
  • Don't forget vendor prefixes for older browsers
