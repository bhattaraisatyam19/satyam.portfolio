HyperText Markup Language

The foundation of every website. Master the structure of the web with semantic, accessible HTML.

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure and content of web pages.

Semantic Structure

HTML provides meaning to web content through semantic elements that describe their purpose.

Hypertext Capabilities

Create connections between documents through hyperlinks, forming the foundation of the web.

Accessibility

Proper HTML structure ensures websites are accessible to all users, including those with disabilities.

HTML Code Examples

Basic Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Document metadata -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
</head>
<body>
    <!-- Visible content -->
    <h1>Hello World</h1>
    <p>This is my first web page.</p>
</body>
</html>

Semantic HTML5

<header>
    <h1>Website Title</h1>
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <h2>Article Title</h2>
        <p>Article content...</p>
    </article>
</main>

<footer>
    <p>© 2023 Company Name</p>
</footer>

Evolution of HTML

1991

HTML 1.0

The first version of HTML with basic text formatting elements.

1995

HTML 2.0

Standardized by IETF with form elements added.

1997

HTML 3.2

W3C recommendation with tables, applets, and text flow around images.

1999

HTML 4.01

Introduced stylesheets, scripting, and accessibility features.

2000

XHTML

HTML reformulated as an XML application with stricter syntax rules.

2014

HTML5

Current major version with semantic elements, multimedia support, and APIs.

HTML Best Practices

Do's

  • Use semantic HTML5 elements
  • Include the lang attribute
  • Write valid, well-formed HTML
  • Use descriptive title tags and meta descriptions
  • Ensure proper nesting of elements
  • Include alt attributes on images

Don'ts

  • Don't use deprecated elements
  • Avoid div soup - use semantic elements
  • Don't skip closing tags
  • Avoid inline styles
  • Don't forget accessibility features
  • Avoid too many nested elements
